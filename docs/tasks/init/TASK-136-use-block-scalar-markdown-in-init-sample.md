## Summary

Make the YAML starter runbook generated by `sw init` use block scalar syntax for
the starter `Markdown.contents` example instead of a string array with a
synthetic empty line.

## Scope

- Update the `sw init` sample Markdown entry to serialize as `contents: |`
- Avoid adding a placeholder blank line solely to force array formatting
- Keep the generated starter runbook valid
- Add regression coverage for the generated YAML shape

## Assumptions

- YAML is the preferred authoring format for starter runbooks.
- The starter Markdown entry should favor the form users are expected to edit by
  hand.
- JSON output may use the equivalent scalar string representation.

## Acceptance Criteria

- [x] Given `sw init --output-file starter.yaml`, the starter Markdown entry is
      serialized with `contents: |`.
- [x] Given `sw init --output-file starter.yaml`, the starter Markdown entry is
      not serialized as a string array with a synthetic empty line.
- [x] Given YAML output from `sw init`, the generated runbook validates
      successfully.
