## Summary

Add a generated-file marker at the top of Markdown produced by `sw run`.

## Scope

- Prepend a fixed HTML comment to generated Markdown output
- Keep the marker as the first line of the generated file
- Use the exact wording defined in `SPEC-003`

## Assumptions

- The marker is a rendering concern only and does not affect validation,
  execution, or assertions.
- The marker wording remains fixed unless the spec is updated intentionally.

## Acceptance Criteria

- [x] Generated Markdown begins with the comment
      `<!-- Generated by Sociable Weaver. Manual edits will be overwritten when regenerated. -->`.
- [x] The generated marker appears before all rendered runbook entries.

## Notes

This task makes generated documentation easier to recognize and clarifies what
will happen to manual edits when the file is regenerated.
